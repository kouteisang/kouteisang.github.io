Redux 安装

Redux 原理

action creator / action

action creator要做的事情就是把我们要做的事情包装成一个action对象

action分为同步action和异步action。

若action是一个同步action，则action只是一个简简单单的js对象，里面存储了要做的事情和要做的数据。

例如: +1，type就是+，data是1。

若action是一个异步action，则action是一个函数。

异步action

若为一个异步action，则action为一个函数，可是我们的reducer不能处理函数，只能处理对象类型的action，因此为了解决这个问题我们需要引入一个中间件。


异步action的返回值是一个函数，该函数中写异步任务，所以这个中间件的作用是帮助我们调用这个函数，等调用完这个函数（异步任务处理完成后），我们再发送一个对象类型的真正的同步action去给reducer处理。

以下代码是一个异步action,这个异步的action需要return一个函数，这个函数被中间件包裹的stroe执行后再分发一个同步action给reducer执行。

dispatch

dispatch将这个action分发出去，交给可以处理的“人”。

store

store是一个调度者，是一个指挥者。（就像饭店老板）。组件通过getState()得到更新好之后的状态。

store并不是自己创建的，我们从redux中获得store，并且store身上有许多的API

reducers

reducers 才是真正对状态进行修改的人。（就像厨师）（饭店老板store是一个调度者，将数据“订单”传递给了reducers，reducers是一个厨师，进行真正数据处理）

Reducers需要接受previousState和action两个参数。比如action是+2，但是reducers如果不知道原来的状态数据，就无法对其+2。

Reducers可以做两件事情，第一件事情是初始化状态，第二件事情是加工状态。如果没有初始化的状态根本没办法进行状态加工。

在进行状态初始化的时候previousState是undefined

在reducers初始化状态的时候,type是@@redux/init,并没有携带data

当我们有多个reducer的时候，我们应该使用combineReducers这个方法将多个reducer结合成一个对象

reducer必须是一个纯函数

  1. 纯函数是一类特别的函数：只要是同样的输入，必定会得到同样的输出（返回）
  2. 不得更改参数数据
  3. 不会产生任何副作用，例如网络请求，输入和输出设备
  4. 不能调用Date.now()或者Math.random()等不纯的方法

react-redux

react-redux原理图

一般UI组件放在component文件夹下，容器组件放在container文件夹下

容器组件不是自己构造的，是通过react-redux进行引入的。

我们只使用redux就已经可以完成状态管理了，那么我们为什么还要引入react-redux呢。这是因为我们曾经使用redux操作redux状态的方法是放在UI组件的例如: store.getState(), store.dispatch()等。但是我们想要做的是UI组件只关注UI本身，不涉及对redux的操作，因此我们使用react-redux，将对redux的操作放在容器组件。UI组件和容器组件的信息传递通过props来实现。

  1. UI组件：不能使用任何redux的api，只负责页面的呈现和交互
  2. 容器组件：负责和redux通信，将结果交给UI组件

react-redux相当于实现了结耦。

  1. 使用connect()()创建一个Count的容器组件
  2. mapStateToProps 传递状态
  3. mapDispatchToProps 传递操作状态的方法
  4. 容器组件的store是靠props传递进去的，而不是在容器组件中直接引入的。

优化

优化一：API层面的优化简写方式

  1. 第一个参数为一个函数，用来传递状态。就是原来的mapStateToProps
  2. 第二个参数为一个对象，对象的value是创建action的方法。这里之所以dispatch没有了是因为我们的react-redux会自动帮我们dispatch。就是原来的mapDispatchToProps。

优化二：

  1. 我们不再需要自己监测状态的改变，即在index.js中我们不需要自己store.subscribe监测状态的改变后进行刷新。这是因为我们的容器组件已经拥有监测状态改变的能力。

  2. store可以直接在入口文件进行传递，Provider从react-redux中获取。

优化三：文件层面的优化

我们在进行开发时，如果一个组件用到了redux那么我们即需要为其编写UI组件又要为其编写container组件。如果基数非常大，那么我们需要多写很多container组件。为了解决这一问题，我们希望把UI组件和container组件进行整合。
