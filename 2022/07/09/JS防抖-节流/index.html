Posted on 2022-07-09 Disqus:
本文详细介绍JS中的防抖与节流

什么是防抖&节流

防抖和节流是JS中常见的性能优化手段，当我们执行一些事件，比如浏览器的resize，scroll; 鼠标的mousemove, mouseover, input输入框的keypress时。如果我们一直不断的调用绑定在事件上的callback函数，就会极大的浪费性能。严重时造成页面的卡顿。所以为了优化性能，我们需要对这类事件进行调用次数的限制。

防抖(debounce)

防抖 : 触发事件一段时间后才执行函数，如果在这段时间内再次触发，则会重新计时。

Debouncing is a programming practice used to ensure that time-consuming tasks do not fire so often, that it stalls the performance of the web page. In other words, it limits the rate at which a function gets invoked.

setTimeout会返回一个timeoutID, 通过clearTimeout清楚指定ID的定时器。

节流

节流：如果猊持续触发事件，每隔一段时间内，只执行一次函数。

异同比较

相同点

  1. 目的都是降低回调的执行频率，节省资源

  2. 都可以使用setTimeout进行实现

不同点

  1. 防抖关注在一定时间内连续触发的事件，只在最后一次执行。节流则侧重于一段时间内只执行一次。

使用场景

防抖

  1. 搜索框搜索输入，在最后一次输入完成后，再次发送请求。
  2. 手机号邮箱的输入验证。
  3. 窗口大小的调整，在调整完成后计算窗口的大小，防止重复渲染

节流的使用场景

  1. 表单的多次点击提交
  2. 滚动加载，加载更多的操作。
