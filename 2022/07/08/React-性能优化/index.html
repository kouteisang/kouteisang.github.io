React 性能优化

Posted on 2022-07-08 Disqus:
本篇博客讲解react的性能优化(Performance turning)

首先我们要知道在哪些情况下一个组件会重新渲染

  1. 一个组件自身的state发生变化

  2. 父组件传递来的props发生了变化

  3. 父组件重新渲染

在react中，每次父组件发生变化其子组件都会重新渲染，无论其自身状态是否发生了变化，因此在一定程度上这是一个开销，我们可以对其进行优化。

在讲解优化之前，应该注意的是当项目未造成瓶颈之前不应该随意优化，因为优化本身也会造成性能开销，不恰当的优化可能造成返效果。

React.memo

对于一个子组件被react.memo包裹，在父组件重新渲染后，子组件会先判断一下其传递来的props是否发生改变，如果发生了改变才会重新渲染这个子组件，否则不进行重新渲染。

React.memo使用浅比较

useMemo

useMemo return a memoized value

  1. useMemo will only recompute the memoized value when one of the dependencies has changed. This optimization helps to avoid expensive calculations on every render.(computeExpensiveValue这个函数会进行很复杂的计算，如果a和b这两个依赖项没进行改变，在render时就不会进行重新计算)

useCallback

前面提到了React.memo()会在父组件传递来的props改变的时候才进行重新渲染

在这种情况下每次父组件重新渲染子组件都会重新渲染，这是因为我们的textFunction方法在每次父组件刷新的时候都会重新创建一个新的，然后传递给子组件，为了解决这个问题我们可以使用useCallBack()这个钩子函数

此时只有当依赖项a改变时才会重新return这个函数，都则每次都是同一个函数，子组件接收到的函数就不会改变，因此就不会重新渲染。
